Java异常详解

Posted on 2018-10-21 | In java , base | | Visitors:
详解java异常，针对异常(Exception)和错误(Error)做详细分析

java 的基本理念是结构不佳的代码不能运行

java 异常继承结构

Java异常通常包括 Exceptions(也即我们最常说的java异常)，Errors(错误)
java中所有异常和错误都继承自Throwable类，也就是说，只有Throwable类以及其子类才能被JVM抛出或者说是能被throw关键字抛出

常用异常及其继承结构

Exception

所有检查异常的父类，所谓检查的异常即在写代码时自己构建的基于业务的异常，我们需要针对这样的异常构建try-catch块，以及时捕获并结合实际场景处理。

RuntimeException

运行时异常。继承自Exception，定义未确认(unchecked)的异常，未检查即代码存在缺陷的不需要catch的不确定异常，标志代码本身存在问题
IOException，IO异常，一些IO操作失败或中断产生的异常

NullPointException

NPE，继承自RuntimeException，最常见的异常，俗称空指针异常，即引用了null类型指针的方法产生的异常。

IllegalArgumentException

继承自RuntimeException，不合法或不合适的参数导致的异常，子类如：NumberFormatException，ClassCastException

ArrayIndexOutOfBoundException

Error

错误类异常，也是一种运行期未确认的异常，标志着正常的程序运行时出现了严重的问题，无法被编译器知道并且导致进程无法恢复或不应该被恢复

VirtualMachineError

StackOverFlowError

栈内存不足导致的错误

OutOfMemoryError

内存溢出导致的异常，如堆内存不足、方法区动态加载时内存溢出等
AssertionError:
AnnotationFormatError:
IOError:

LinkageError

被依赖类在依赖类编译后发生了不兼容的改变，导致调用失败
